🔐 Auth Service Deep Dive

How Login Works

Users can sign up with email/password or via OAuth2 (Google, Discord). Passwords are hashed with bcrypt (12 rounds). On login, the server issues two tokens:

  • Access Token — JWT, short-lived (15 minutes), stored in memory (not localStorage)
  • Refresh Token — long-lived (30 days), stored in an httpOnly cookie, rotated on every use

OAuth2 Flow

User clicks Discord
Redirect to Discord OAuth
Discord returns code
Server exchanges for token
Create/link user in DB
Issue JWT pair

2FA Implementation

Use speakeasy (Node.js) to generate TOTP secrets. User scans QR with Google Authenticator. On each login, server validates the 6-digit code with a 30-second window. Backup codes (10 one-time codes) are generated and stored hashed.

Security Rules

  • Never store tokens in localStorage — XSS risk
  • All API endpoints require valid JWT in Authorization header
  • Failed login attempts trigger exponential backoff (5 attempts = 15 min lockout)
  • All passwords stored as bcrypt hashes — never plain text
📝 Post & Feed Service Deep Dive

Database Schema (PostgreSQL)

  • posts — id, user_id, content, type, tags[], image_url, created_at
  • post_likes — post_id, user_id, created_at (unique pair)
  • post_views — post_id, viewer_id, viewed_at (deduplicated per 24h)
  • comments — id, post_id, user_id, content, parent_id

Feed Algorithm

The home feed is scored per-post: score = (likes × 3) + (comments × 5) + (views × 0.1) + (recency_bonus). Posts from followed users get a 2× boost. Redis caches the top 100 feed posts per user for 5 minutes.

Image Upload Flow

User selects image
Client requests upload URL
Server validates & signs S3 URL
Client uploads directly to S3
Cloudinary processes & CDN-serves

Aura Point Triggers

  • Post published: +5 pts
  • Per unique view (after 3 seconds): +0.1 pts
  • Like received: +2 pts
  • Comment received: +3 pts
  • Tip received: +10 pts + coin value
🏆 Ranking Service Deep Dive

Redis Sorted Set for Leaderboard

Redis ZADD leaderboard:weekly [score] [user_id] updates in real time. ZREVRANGE retrieves the top-N users in O(log N). This handles millions of users with sub-millisecond reads.

Tier Logic

  • Bronze: 0 – 999 pts
  • Silver: 1,000 – 4,999 pts → unlock custom profile tags
  • Gold: 5,000 – 19,999 pts → unlock event priority + profile badge
  • Platinum: 20,000 – 99,999 pts → unlock studio access + higher visibility
  • Legend: 100,000+ pts → unlock cash-out + creator fund eligibility

Rank Update Triggers

Engagement event fires
Post service emits event
Rank service consumes queue
ZINCRBY in Redis
Check tier threshold
Push notification if promoted

Anti-Fraud

  • View deduplication: same IP + user_id only counts once per 24h per post
  • Bot detection: views under 3 seconds session time are discarded
  • Point velocity check: users gaining >10,000 pts/hour are flagged for review
💰 Reward / Coin Service Deep Dive

The Ledger Principle

Every AniCoin transaction is an immutable record. We never modify a balance directly — we INSERT a new ledger row and SUM to get balance. This gives a full audit trail and prevents fraud.

Ledger Schema

  • transactions — id, from_user_id, to_user_id, amount, type, fee, created_at, reference_id
  • Types: earn_view, earn_event, tip_sent, tip_received, withdrawal
  • Balance = SUM(credit_amount) - SUM(debit_amount) for a user

Tip Transaction Flow

User taps Tip button
Frontend sends POST /tip
Server checks sender balance
BEGIN TRANSACTION
Debit sender, credit creator (−8% fee)
COMMIT + notify both users

Cash-Out (Legend Only)

Razorpay Payout API is used for UPI transfers to India. Minimum withdrawal: 1,000 AniCoins = ₹100. Withdrawal is processed within 2 business days. KYC verification required (Aadhaar + PAN for amounts over ₹10,000).

Platform Revenue from Coins

  • 8% fee on every tip transaction
  • AniPro subscription: ₹149/month (badges, no ads, 2× view rewards)
  • Event entry sponsorships from studios
🎉 Events Service Deep Dive

Event Lifecycle

Admin creates event
Published to feed
Users submit entries
Community votes open
Deadline hits (cron job)
Winners auto-paid in AniCoins

Database Schema

  • events — id, title, description, type, prize_pool, start_at, end_at, sponsor_id
  • event_entries — id, event_id, user_id, post_id, submitted_at, vote_count
  • event_votes — event_id, voter_id, entry_id, voted_at (one vote per voter per event)

Real-Time Live Events

Socket.io powers live vote counts, live entry submissions, and the countdown timer. When a vote is cast: socket.emit('vote_update', {entry_id, new_count}) is broadcast to all clients watching that event room.

Sponsored Events

Studios pay to create branded events. They set the prize pool (in AniCoins or real cash), category rules, and optionally a jury panel. AnimeVerse charges ₹15,000–₹1,00,000 per sponsored event depending on prize size and platform featuring.

🛡️ Security Layer Deep Dive

API Rate Limiting

  • General API: 100 requests/minute per IP
  • Auth endpoints: 5 attempts/15 minutes (prevents brute force)
  • Upload endpoint: 10 uploads/hour per user
  • Tip endpoint: 20 tips/hour per user (prevents automated draining)

Content Security

  • All image uploads scanned by AWS Rekognition for NSFW content before serving
  • Post text sanitized with DOMPurify to prevent XSS
  • SQL injection prevented by parameterized queries (never string concatenation)
  • File type validation: only JPG, PNG, GIF, WebP accepted (checked by magic bytes, not extension)

Data Security

  • All data in transit: HTTPS (TLS 1.3)
  • Passwords: bcrypt hash (never stored plain)
  • Database: encrypted at rest (AWS RDS with AES-256)
  • Sensitive fields (phone, UPI ID): encrypted at application level before storing
  • GDPR-compliant: users can export or delete all their data

Infrastructure

  • Cloudflare in front of everything: DDoS protection + WAF
  • VPC: backend services not publicly accessible, only through API gateway
  • Secrets management: AWS Secrets Manager (never .env files in production)
  • Logging: all auth events, transactions, and moderation actions logged to CloudWatch

Coin Fraud Prevention

  • All coin operations are atomic DB transactions — no double-spend possible
  • Withdrawal requires email OTP confirmation
  • KYC required for cash-out (Aadhaar + PAN stored encrypted)
  • Velocity limits: max 5,000 AniCoins withdrawable per week
